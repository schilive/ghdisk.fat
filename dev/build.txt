# How to Build the Project
## Introduction
Requirements:
        - Windows (>= Windows 2000) or Unix.
        - GNU Make; extensions are used.
        - GCC/Clang or MSVC with Developer Command Prompt, with at least C89.
        - Optional: Python (>= 3.0) for translation support.

Supported translations:
        - Brazilian Portuguese, code 'pt'.

To build with default options, use just 'make' or 'make
build'. The default compiler toolchain is GCC, the host OS is auto-detected,
and the project is built on auto-made directory 'build'.

Everything generated by the project is put inside the "build directory", by
default "build". The output executable is located at 'build/$(M_TARGET)' with
executable file extension added at the end.

## Targets
There are two build Makefile targets.
        - 'build' (default): it builds the project with the given options.
        - 'clean': it deletes the build directory.

## Options
All build options start with "M_", and are regular Makefile options. So, they
can be set through the Make tool, such as "make M_TARGET=target", or through
an environmental variable.

### File Options
- 'M_TARGET = ghdisk.fat': the name of the output binary executable. The
  OS-specific executable file extension is added by Make, so it should not be
  provided in this option value.
- 'M_BUILD_DIR = build': the build directory, created automatically if not
  present. If a non-directory file of the same name exists, an error is
  raised.

### OS Options
- 'M_HOST_OS [= WIN32 | NIX]': if not set manually, auto-detected, first
  through whether the environmental variable 'OS' value is 'Windows_NT', else
  whether issuing the command 'uname' returns exit code 0, else an error is
  raised. Its choice determines the tools used (e.g. 'rm' or 'DEL'), the file
  extensions used (e.g. ".o" or ".obj"), and the default Python binary name
  ('python3' or 'python').

### Compiler Toolchain Options
- 'M_COMPILER_MSVC = 0': if set, it uses the compiler toolchain
  and interface compatible with MSVC, otherwise compatible with GCC.
- 'M_CC = gcc | CL': the C compiler. If 'M_COMPILER_MSVC = 0', then 'M_CC' is
  also used for linking and 'gcc' is the default, otherwise 'CL' is default.
- 'M_CFLAGS = -Wall -Wextra -pedantic -std=c89 | /Wall': the C compiler flags
  added to every use of 'M_CC'. If 'M_COMPILER_MSVC = 0', then the first list
  of flags is default, otherwise the second.
- 'M_ACFLAGS': like 'M_CFLAGS', but never modified internally, and added
  after.
- 'M_LD = LINK': if 'M_COMPILER_MSVC' is set, then 'M_LD' is the linker.
- 'M_LFLAGS': the linker flags added to every use of 'M_LD'.
- 'M_ALFLAGS': like 'M_LFLAGS', but never modified internally, and added after.

- 'M_DEBUG = 0': if set, the debug flag is added to 'M_CFLAGS' and 'M_LFLAGS'.
- 'M_OPTIMISE = 0': if set, the optimisation flag of most aggressive
  optmisation, except those that may change behaviour, are added to 'M_CFLAGS'
  and 'M_LFLAGS'.

- 'M_PYTHON = python3 | python': the Python interpreter used.

### System-Interface Options
- 'M_SYS = stdc | nt': it sets which system interface is used; "stdc" for ISO
  C interface or "nt" for Windows NT interface. By default, it chooses "stdc".
  However, it is highly recommended to use the option specifically for your
  OS.

### Translation Options
- 'M_LANG': if set, the translation identifier code. It uses the translation
  in directory 'po/M_LANG/' The available translations and their codes are
  listed in the introduction.
